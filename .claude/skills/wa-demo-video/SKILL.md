# wa-demo-video — WhatsApp Chat Emulator for Demo Recording

A local WhatsApp-styled chat emulator that renders a pixel-perfect WhatsApp interface.
The AI responds using the lead's system prompt from Google Sheets, creating realistic
demo conversations for screen recording.

## Quick Start

### 1. Start the emulator server

```bash
# Check if already running
curl -s http://localhost:8889/ > /dev/null 2>&1 && echo "running" || echo "not running"

# Start if not running
python3 /Users/devlink007/WADemoSender/.claude/skills/wa-demo-video/emulator/server.py > /tmp/wa_emulator.log 2>&1 &
echo $! > /tmp/wa_emulator.pid
sleep 2
curl -s http://localhost:8889/ > /dev/null 2>&1 && echo "Server started on port 8889" || echo "ERROR: check /tmp/wa_emulator.log"
```

### 2. Open an emulator link

Links are generated by the `/wa-emulator-link` skill. Format:
```
http://localhost:8889/?name={BusinessName}&phone={phone}&spreadsheet={ID}&row={N}
```

The emulator auto-loads:
- Business name in the WhatsApp header
- Lead's avatar from `output/wa_avatars/{phone}.jpg` (if available)
- System prompt from Google Sheets (configures AI behavior)

### 3. Record the demo

1. Open the emulator link in browser
2. Chat with the AI — it responds as the prospect's WhatsApp assistant
3. Record the screen (OBS, QuickTime, or any screen recorder)
4. Save recording and paste the direct video URL into "WhatsApp Demo" column

## Architecture

The emulator consists of:

- **server.py** — Python HTTP server on port 8889
  - Serves static files (HTML/CSS/JS)
  - `/proxy/chat` — Proxies chat messages to winbixai.com AI API
  - `/proxy/configure-prompt` — Injects the lead's system prompt from Google Sheets
  - `/proxy/wa-avatar` — Serves cached WhatsApp profile pictures

- **index.html** — WhatsApp chat interface layout
- **styles.css** — Pixel-perfect WhatsApp dark theme styling
- **app.js** — Chat logic, message rendering, API communication
- **doodle-dark.svg** — WhatsApp background pattern

## Demo Response Guidelines

The AI in the emulator follows the lead's system prompt, which defines:
- Persona (responds AS the business owner)
- Services and pricing
- Qualification flow (qualify → book a call)
- Tone (casual, 1-2 sentences max)

## Configuration

The emulator reads from:
- **Google Sheets** — System prompt, business name (via spreadsheet/row URL params)
- **output/wa_avatars/** — Cached WhatsApp profile pictures
- **winbixai.com** — AI backend API

## Port

- WhatsApp emulator: **8889**
- Instagram emulator (if running): **8888** (separate project, both can run simultaneously)

## Troubleshooting

- **Server won't start**: Check if port 8889 is already in use (`lsof -i :8889`)
- **AI not responding**: Verify winbixai.com is reachable and system prompt is loaded
- **No avatar showing**: Run avatar fetch from `/wa-emulator-link` skill first
- **System prompt not loading**: Ensure the spreadsheet is shared with the service account
